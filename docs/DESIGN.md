# 系统设计文档

## 项目概述

本项目是一套完整的人脸识别签到系统，支持手机端拍照上传进行签到，服务器端完成人脸识别与身份匹配。系统基于华为云FRS服务实现核心人脸识别功能。

│ ┌──────────────┐ ┌──────────────┐ │
│ │ MySQL │ │ Redis │ │
│ │ (用户/记录) │ │ (缓存/限流) │ │

## 模块设计

### 用户模块
- 用户注册、登录、注销
- JWT Token维护会话
- 密码加密存储（bcrypt）

### 人脸库模块
- 用户注册时将人脸照片添加到FRS人脸库
- 支持人脸库管理（添加、删除）
- 人脸ID与用户ID关联

### 签到模块
- 接收签到照片
- 调用FRS人脸搜索接口识别用户
- 相似度阈值验证（建议80%）
- 签到限流处理（5秒内不重复签到）

### 记录模块
- 查询个人签到记录
- 支持按时间筛选
- 签到统计分析
- 管理员查看所有记录

## 核心流程

### 用户注册流程

1. 用户在App填写基本信息，拍照或选择标准人脸照片
2. App将信息+照片发送到服务器
3. 服务器将用户信息存入MySQL
4. 服务器调用FRS add_face接口，将照片添加到人脸库
5. 将FRS返回的face_id存入MySQL用户表

### 签到流程

1. 用户打开App拍照
2. App压缩图片后上传到服务器签到接口
3. 服务器调用FRS search_face接口，在人脸库中搜索匹配用户
4. 返回匹配结果（用户ID + 相似度）
5. 校验相似度是否超过阈值（建议80%），若超过则记录签到
6. 返回签到成功/失败信息给App

## 高并发处理

### 限流
- 同一用户5秒内不能重复签到
- 使用Redis实现限流

### 削峰
- 签到请求先进入Redis缓存
- 异步处理FRS调用

### 缓存
- 使用Redis缓存当日已签到用户
- 避免重复数据库写入

### 重试
- FRS调用失败时自动重试3次

## 技术选型

### 后端
- **Python 3.10+**: 语法简洁，生态丰富
- **FastAPI**: 高性能异步框架，自动生成API文档
- **MySQL 8.0**: 存储用户信息、签到记录
- **Redis 7.0**: 存储Token、限流计数器
- **华为云FRS SDK**: 提供人脸检测、人脸库管理、人脸搜索能力

### 前端
- **uni-app/原生Android**: 跨平台开发或原生开发
- **Axios**: HTTP请求库
- **相机API + 图片压缩库**: 拍照并压缩上传

### 部署
- **华为云ECS**: 2核4G，Ubuntu 22.04
- **Nginx**: 反向代理+静态文件
- **Supervisor/systemd**: 进程管理

## 安全考虑

- 密码使用bcrypt加密
- JWT Token用于API认证
- HTTPS通信（生产环境）
- 请求限流防止滥用
- 输入验证和清理
- SQL注入防护（使用ORM）

## 性能指标

- 支持并发签到：100+/秒
- 平均响应时间：< 2秒
- 数据库连接池：10-30
- 缓存命中率：> 80%
